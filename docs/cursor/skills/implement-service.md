# implement-service Skill

## Purpose

Scaffold a backend service with API, DB, and business logic.

## When to Use

- "Implement a new API for X"
- "Create a service that does Y"
- "Add an endpoint for Z"

## Steps

1. Ensure architecture/spec exists (Architect or API contract).
2. Create service file (e.g. `services/x.service.ts`).
3. Add API route/controller.
4. Add DB model/migration if needed (Prisma).
5. Add unit tests.
6. Add integration test for the endpoint.
